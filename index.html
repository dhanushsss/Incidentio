What Zerqis Does

Detect, track, alert, and rotate—all in one platform

🔍

Detect Secrets

Scans application logs, CI/CD pipelines, and optionally Git commits and cloud configs for exposed secrets (AWS keys, JWTs, passwords, DB URLs).

📋

Track in Inventory

Central secret inventory with risk scoring. Filter by service, environment; see exposures and rotation events in one place.

🔔

Alert Your Team

Slack and generic webhook alerts when exposures are found. Formatted messages: Service, Environment, Secret Type, Severity.

🔄

Support Rotation

Rotate compromised secrets via API. Pluggable rotators with AWS implementation; trigger rotation from the dashboard or automation.

Features

Everything you need to get from sign-up to detecting exposures

📥

Log Ingestion

  • POST /api/v1/logs with X-API-Key; JSON: serviceName, environment, logMessage
  • Logs streamed to Kafka; processed asynchronously; 202 Accepted
  • Onboarding page with sample log; Integration Guide (cURL, Node.js, Python); optional Log Agent for Docker/Kubernetes
🔐

Secret Detection

  • Regex and rule-based scanning: AWS keys, JWT, Stripe, postgres://, password=, and more
  • Git webhook: GitHub and GitLab push events; scan commit content; same detection rules
  • Cloud discovery (optional): AWS IAM keys, Kubernetes Secret resources via GET /api/v1/secrets/discovered
📊

Dashboard & Risk

  • Overview: total secrets, exposures, risk score; Secret Inventory; Exposure Events; Security Risk breakdown
  • Usage: logs processed, secrets detected, exposures by service; risk trend
  • Risk score by production exposure, secret type, frequency; prioritize remediation
🔑

API Keys & Onboarding

  • Create/revoke API keys in dashboard (Organization → API Keys); raw key shown once; use in X-API-Key for ingestion
  • GET /api/v1/onboarding: steps + curl/nodejs/python examples; dashboard Onboarding and Integration Guide
  • No credit card for standard SaaS trial; provide email, password, display name, optional org name

Security & Multi-Tenancy

RBAC, tenant isolation, and secure ingestion

👥

Roles & Access

  • ADMIN: tenant management, system metrics, alert history
  • SECURITY_ENGINEER: full access to secrets, exposures, rotations, risk, API keys
  • VIEWER: read-only dashboards and reports
  • JWT for dashboard; API key for log ingestion only—never use the same key for both
🏢

Tenant Isolation

  • All data scoped by tenantId: User, Secret, SecretExposure, RotationEvent, ApiKey
  • TenantContext from JWT or X-API-Key; gateway resolves and sets X-Tenant-Id
  • Only users and API keys belonging to your org can see or send data for your tenant
🔒

Auth & Gateway

  • Register: email, password, displayName; Login returns accessToken (Bearer); use for all dashboard APIs
  • Gateway: JWT validation, rate limiting, security headers (HSTS, X-Frame-Options, CSP, etc.)
  • API keys: 32-byte secure keys; only hash stored; never log raw secrets; audit logging
📡

Alerts & Rotation API

  • Slack webhook and generic webhook; set SLACK_WEBHOOK_URL / ALERT_WEBHOOK_URL
  • POST /api/v1/rotations/trigger: secretId, secretType, oldIdentifier (e.g. AWS key ID); Rotation Service runs rotator (e.g. AWS)
  • OpenAPI/Swagger at /swagger-ui.html and /v3/api-docs per service

How It Works

Architecture and data flow

Ingestion → Detection

Client sends POST /api/v1/logs with X-API-Key. Gateway forwards to Ingestion Service; logs published to Kafka topic logs-ingestion. Detection Engine consumes logs, runs regex rules, creates/updates Secret and SecretExposure in PostgreSQL, publishes to secret-exposure-alerts.

Services

API Gateway (8080), Auth (8081), Ingestion (8082), Detection Engine (8083), Inventory (8084), Rotation (8085), Alert (8086), Git Scanner (8087), Cloud Discovery (8088). React dashboard (Vite + TypeScript) talks to gateway; JWT and API key resolve to tenant.

Observability

Each service: /actuator/health, /actuator/prometheus. Metrics: detection_logs_processed_total, detection_secrets_detected_total, secret_rotations_total, tenant_logs_processed_total, ingestion_rate_limited_total. Grafana can scrape for dashboards.
