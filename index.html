Technical reference

Stack, setup, API — expand when you need it

Languages & stack

Languages (typical): Java (Spring), JavaScript/TypeScript (Express-style), Python (Flask/FastAPI), Ruby, Go — plus fetch/axios-style frontend calls where detected.

Runtime: Java 21, Spring Boot 3.2, PostgreSQL, Flyway, React (Vite). Scans use a DB-backed queue; PR jobs can preempt manual scans.

Setup & environment

GitHub OAuth app: Homepage http://localhost:3000 · Callback http://localhost:8080/api/auth/callback

Required: GITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET, JWT_SECRET, DB_URL, DB_USERNAME, DB_PASSWORD

Optional: FRONTEND_URL, GITHUB_WEBHOOK_SECRET, ANTHROPIC_API_KEY, SLACK_WEBHOOK_URL, PR limits and status flags. Browser token: zerqis_token (legacy architect_token migrates on sign-in).

API overview

Base URL example http://localhost:8080. App API under /api/v1. Authenticated routes: Authorization: Bearer <jwt>.

Public: GET /api/auth/github, GET /api/auth/callback, POST /api/webhooks/github, GET /api/public/version, GET /api/public/v1/health, GET /api/public/v1/graph/{owner} with X-API-Key.

JWT areas: /api/auth (e.g. GET /me), /api/v1/repos, /api/v1/scan, /api/v1/graph, /api/v1/insights, /api/v1/impact, /api/v1/ai, /api/v1/enterprise, /api/v1/keys, /api/v1/dashboard. Scan triggers may return QUEUED while the queue runs.

PR webhook & configuration

POST /api/webhooks/github for pull_request (opened, synchronize). Optional HMAC via GITHUB_WEBHOOK_SECRET. Work runs async after 200. Connect the repo and run at least one deep scan first so callers exist in the DB.

  • PR_MAX_FILES_SCAN (default 50)
  • PR_POST_COMMIT_STATUS / PR_FAIL_ON_REVIEW
  • FRONTEND_URL — links in comments
  • ANTHROPIC_API_KEY + PR_AI_RISK_TIMEOUT_MS — optional AI Insight text only
