What Zerqis Does

Detect, track, alert, and rotate—all in one platform

🔍

Detect Secrets

Scans application logs, CI/CD pipelines, and optionally Git commits and cloud configs for exposed secrets (AWS keys, JWTs, passwords, DB URLs).

📋

Track in Inventory

Central secret inventory with risk scoring. Filter by service, environment; see exposures and rotation events in one place.

🔔

Alert Your Team

Slack and generic webhook alerts when exposures are found. Formatted messages: Service, Environment, Secret Type, Severity.

🔄

Automated Rotation

Pluggable SecretRotator (AWS IAM, Vault stub): trigger → disable old → create new → store event. Trigger via API or dashboard.

💥

Blast Radius Analysis

For exposed secrets (e.g. AWS_ACCESS_KEY), answer what can this secret access? — IAM user, permissions, affected resources (S3, DynamoDB, Lambda); dashboard and API.

Features

Everything you need to get from sign-up to detecting exposures

📥

Log Ingestion

  • POST /api/v1/logs with X-API-Key; JSON: serviceName, environment, logMessage
  • Logs streamed via Kafka; 202 Accepted; optional Fluent Bit, Logstash, or Log Agent (Docker/Kubernetes)
  • Onboarding + Integration Guide (cURL, Node.js, Python); no credit card for standard trial
🔐

Runtime Secret Detection

  • Regex and rule-based scanning: AWS keys, JWT, Stripe, postgres://, password=, and more
  • Git webhook: GitHub and GitLab push events; scan commit content; same rules; produce to Kafka
  • Cloud discovery (optional): AWS IAM keys, Kubernetes Secret resources — GET /api/v1/secrets/discovered
💥

Blast Radius & Risk

  • Blast radius: per exposure — what can this secret access? IAM user, permissions, affected resource types (S3, DynamoDB, Lambda); dashboard and GET /api/v1/blast-radius, /api/v1/exposures/{id}/blast-radius
  • Risk scoring engine: overall score and level from production exposures, secret sensitivity, frequency, unique secrets
  • Dashboard: Overview, Secret Inventory, Exposure Events, Blast Radius (per exposure), Rotations, Risk, Usage
📋

Inventory & Alerting

  • Track Secret, SecretExposure, RotationEvent by tenant; firstSeen, lastSeen, exposure count
  • Alerting: Slack webhook, generic webhook, SIEM-style event forwarding; severity LOW/MEDIUM/HIGH/CRITICAL
  • API keys: POST/GET/DELETE /api/api-keys; 32-byte keys, hash stored; X-API-Key for POST /api/logs

Security & Multi-Tenancy

RBAC, tenant isolation, and secure ingestion

👥

Roles & Access

  • ADMIN: tenant management, system metrics, alert history
  • SECURITY_ENGINEER: full access to secrets, exposures, rotations, risk, API keys
  • VIEWER: read-only dashboards and reports
  • JWT for dashboard; API key for log ingestion only—never use the same key for both
🏢

Tenant Isolation

  • All data scoped by tenantId: User, Secret, SecretExposure, RotationEvent, ApiKey
  • Designed to support 50+ tenants from day one with full isolation
  • TenantContext from JWT or X-API-Key; only your org’s users and keys see your data
🔒

Auth & Gateway

  • Register: email, password, displayName; Login returns accessToken (Bearer); use for all dashboard APIs
  • Gateway: JWT validation, rate limiting, security headers (HSTS, X-Frame-Options, CSP, etc.)
  • API keys: 32-byte secure keys; only hash stored; never log raw secrets; audit logging
📡

Alerts & Rotation API

  • Slack and generic webhook; SIEM-style event forwarding; severity LOW/MEDIUM/HIGH/CRITICAL
  • POST /api/v1/rotations/trigger: secretId, secretType, oldIdentifier; rotator disables old, creates new, stores event
  • Swagger UI at /swagger-ui.html, OpenAPI at /v3/api-docs per service

How It Works

Architecture and data flow

Ingestion → Detection

Client sends POST /api/v1/logs with X-API-Key. Gateway forwards to Ingestion Service; logs published to Kafka topic logs-ingestion. Detection Engine consumes logs, runs regex rules, creates/updates Secret and SecretExposure in PostgreSQL, publishes to secret-exposure-alerts.

Services

API Gateway (8080), Auth (8081), Ingestion (8082), Detection Engine (8083), Inventory (8084), Rotation (8085), Alert (8086), Git Scanner (8087), Cloud Discovery (8088), Secret Graph (blast radius). React dashboard (Vite + TypeScript) talks to gateway; JWT and API key resolve to tenant.

Observability

Each service: /actuator/health, /actuator/prometheus. Metrics: detection_logs_processed_total, detection_secrets_detected_total, secret_rotations_total, tenant_logs_processed_total, ingestion_rate_limited_total. Grafana can scrape for dashboards.
