Features

End-to-end FinOps capabilities for Kubernetes and cloud

☸️

Visibility & Cost Attribution

  • Kubernetes cost from workload metrics (CPU/memory, replicas) with configurable rates
  • AWS Cost Explorer integration; daily snapshots; tag-to-namespace mapping
  • Unified executive view: combined K8s + AWS spend and trends (30/60/90 days)
  • Cloud accounts page: linked AWS (GCP/Azure later), cost by service, period filters
📉

Cost Analytics & Reporting

  • Cost by cluster, namespace, deployment; date range and filters; CSV export
  • Executive dashboard: spend, trend chart, top namespaces, budget usage, waste %, anomalies
  • Tenant-branded monthly PDF report (year-month picker); audited on generate/download
🔧

Optimization & Waste Reduction

  • OptimizationEngine: 7-day workload analysis → right-sizing and waste recommendations
  • Types: CPU_RIGHT_SIZE, MEMORY_RIGHT_SIZE, IDLE_NAMESPACE, ZOMBIE_SERVICE, UNUSED_PVC, REPLICA_OPTIMIZATION
  • Optimization dashboard: filter by cluster/namespace; mark APPLIED/DISMISSED; potential savings & waste %
🔔

Budget & Alerts

  • Per-tenant monthly budget; configurable 70% and 90% alert thresholds
  • Email (alert_email) and Slack webhook notifications; scheduled daily check (e.g. 9 AM)
  • Statistical anomaly detection on cost; daily job; anomalies on Executive dashboard

Multi-Tenancy & Security

Roles, isolation, audit, and secure ingestion

👥

Roles & Access Control

  • SUPER_ADMIN: manage tenants, full audit
  • TENANT_ADMIN: clusters, AWS config, budgets, reports
  • TENANT_VIEWER: read-only dashboards and reports
  • All data scoped by tenant_id; context from JWT or X-Tenant-Id (super admin)
📋

Audit & Compliance

  • Events: LOGIN, REFRESH_TOKEN, REPORT_GENERATED, REPORT_DOWNLOAD, BUDGET_WARNING_SENT, CLUSTER_CREATED, TOKEN_ROTATED, etc.
  • Paginated Audit page; super admin can filter by tenant
🔐

Auth & Secrets

  • JWT access + refresh tokens; configurable TTL; password policy; rate limiting (per IP/user; 429 when exceeded)
  • Kubeconfig encrypted at rest (AES-256-GCM); never returned in API (masked); ingestion token rotation via API
  • Push-based token-authenticated ingestion; AWS via DefaultCredentialsProvider / IRSA—no long-lived keys
📥

Data Ingestion & Jobs

  • Agent (Helm CronJob) in cluster → Metrics API → aggregate per deployment → POST to InfraSight
  • Scheduled jobs: cost calc (2 AM), AWS sync (3 AM), retention (4 AM), anomaly (5 AM), report (6 AM 1st), budget (9 AM)
  • One Cost Explorer request per tenant per day; optional “Sync AWS now” and cost-by-service from UI

How Efficient It Is

Architecture, performance, and scalability

Stack & Performance

Java 21, Spring Boot 3, PostgreSQL, React 18 + TypeScript + Vite. Dedicated cron pool for scheduled jobs; @Async for audit and heavy work. Redis-backed rate limiter for multi-instance deployments.

Cost & API Efficiency

Daily AWS sync = one Cost Explorer call per tenant per day; cost-by-service on-demand. Push ingestion: one POST per cluster per run. Cost calculation once per day over stored metrics—no heavy queries on every dashboard load.

Scalability & Ops

Horizontal scaling with Redis rate limit and optional Kafka for ingestion. DataRetentionJob purges old snapshots. Actuator health, Prometheus-ready. Cost accuracy validated (e.g. ≤10% variance vs AWS) before customer use.
